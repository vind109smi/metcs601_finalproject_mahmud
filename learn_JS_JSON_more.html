What Is There To Know About Javascript?

I have learned that Javascript is the most widely used web programming lanage that helps me combine web content with functionalities, similar to regular OOP. Javascript helps with HTML, CSS and even server side technology such as Node.js. It all started off with ECMAScript standard back in 1997 when it declared JS to be an official language. Javascript interacts well with the DOM where we use DOM as API for XML documents. The Browser Object Model or BOM is for methods and interfaces for accessing and manipulating web browser outside of context. Just like CSS, we can either write code using internal JS inside HTML by denoting the script tag and adding our JS magic there, or better yet, when I do large scale complex web coding, I need to apply external JS which is via the link to the external file. What is neat is that even in my daily profession and in this course, I can debug Javascript using developer tools such as Chrome Debugger. This allows me to spot any javascript errors, read upon it along with the line it points to the error, and perform other handy testing tricks such as console.log() an object, writing JS code in console and so much more. Throughout this module, I have learned different things of JS such as statements, expressions, variables, inputs and outputs such as alerts, document.write, etc., data types, conditionals, functions, and even most importantly to me, scoping. JSON or Jason?

In case you are wondering, JSON stands for JavaScript Object Notation, but the way we say it sounds like the name 'Jason'. If we say slowly, then yeah, we notice the difference. Meh. JSON is pure essence of organizing my content in contained format such as arrays. This is pure data interchange. I have applied JSON in my fifth assignment. I applied collection of name/value pairs that represent objects and we have ordered list of values in array format, such as: { "my_favorite_foods" : [ { "fast_food": {"foodName": "Ramen", "type": "noodles", "origin": "Japanese" } ... } ] }. I even learned FETCH, no I don't mean play fetch with my doggie. Fetch is a clean built in API that helps me quickly fetch and handle data on server. I learned handy tricks with Fetch such as applying promises (Module 5, 2023).

Vue.Js: The Coating to Great JS Development

Since this module was short, I applied Vue internal to the HTML. I learned things such as binding attributes such as v-bind, v-on, and other neat attributes such as for class styling, event handling and conditions. One thing I learned, we need to be careful how we code the Vue component. If we are using Vue2, we need to do the const app = new Vue() instantiation and use el: '#id' and if we use Vue3, then we do the const app = Vue.createApp().mount("#id)". In between is how we return the data. The Vue2 can be done via function and object, but Vue3 is strictly for the function. We grab the Vue functionality through CDN. There are many sources out there but I am careful to choose the right version and the mininification js one (Module 6, 2023).
