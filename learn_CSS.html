Important Things to Known About CSS3


Effects of Cascading

What I have learned when reviewing the CSS3 modules is that cascading is a way for us to make our HTML look pretty. We need to found out where the CSS styling file or items can be found. We do not want to deal with conflicts or redundancies. There are three types of cascading that I have learned while applying them in my assignments, two of them prevalently used.
1. Inline styles -> This is where I do all my styling within one single HTML element. Ugghh, this is not efficient.
2. Internal styles -> This one is what I use in some of my pages that have unique styling, even in this project, where I can apply the style tag inside my head element section. Not fully recommended to include all the time, but sparingly.
3. External styles -> This is what I am talking about. Throughout my course assignments and even in this project, I have and am using global styles where I have one common styling page and depending on the link reference I am providing on the specific html page, will active the cascading styles. This is also deemed as a separate CSS file. (Module 2, 2023)

Utilizing CSS with DOM

Something that I have learned in the course modules is that we can apply DOM to activate styling rules. Not much I have done so far in my assignments with complex rules but something I found interesting. Say we have parent elements. They too have child elements. Say div parent element that contains the paragraph child element. We can define a styling rule to the direct descendant to the div element. This is known as a nested rule. E.g. take "div > p { margin: 5px}". This will take place for every paragraph element contained within a div element that will have 5px margin. We can go deep into the DOM tree as much as we want, just not too crazy. The truth is, I applied ids to my individual elements and did unique styling so I never resorted to this. But I could have. (Module 2, 2023)

Windows Progressive Apps

One thing that I have learned and contributed in my discussions was the use of progressive apps. The first summary point is about progressive web apps themselves. The issue we face is that various mobile devices and virtual keyboards that we use make it harder for us to manage content. What is worse, we face the same roadblocks with dual screen computers or foldable devices (Brosset, 2021). Thankfully, progressive web apps help us tackle this nuisance, where we can find these types of apps "stable, linkable, searchable, and responsive just like websites". Not to mention, they can work in offline mode and read files (Brosset, 2021). We can apply the styling techniques in between web and device native interfaces. But, the problem is that progressive web apps have the rectangular constricted windows with the generic title bars at the very top (Brosset, 2021).
